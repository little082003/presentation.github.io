📋 Requirement คืออะไร?

Software Requirement = ข้อกำหนดที่ระบุว่าซอฟต์แวร์ "ต้องทำอะไร" และ "ทำได้ดีแค่ไหน"
เป็นพื้นฐานที่สำคัญที่สุดในการออกแบบ Test Case

✅ Functional Requirement (FR)

ระบบ ต้องทำ อะไรได้บ้าง
เช่น: ค้นหาสินค้าตามชื่อ, แสดงผลลัพธ์

⚡ Non-Functional Requirement (NFR)

ระบบทำงาน ได้ดีแค่ไหน
เช่น: แสดงผลภายใน 2 วินาที, รองรับ 10,000 users

📖 อ้างอิง: IEEE 830 / ISO/IEC/IEEE 29148:2018 — มาตรฐานการเขียน SRS

เป้าหมาย: ให้ลูกค้าหาสินค้าเจอ → ซื้อ (Conversion)

📐 เทคนิคการออกแบบ Test Case (ISTQB)

1. Equivalence Partitioning (EP)

แบ่งกลุ่มข้อมูล Input ที่ให้ผลลัพธ์เหมือนกัน

  • Valid คำค้นหา 1-50 ตัวอักษร → แสดงผล
  • Invalid ค่าว่าง (0 ตัวอักษร) → Error
  • Invalid เกิน 50 ตัวอักษร → Error
2. Boundary Value Analysis (BVA)

ทดสอบค่าที่ขอบเขต เพราะ Bug มักเกิดตรงนี้

0 ตัวอักษร Invalid
1 ตัวอักษร Min Valid
50 ตัวอักษร Max Valid
51 ตัวอักษร Invalid

📖 อ้างอิง: ISTQB Foundation Level Syllabus v4.0 — Chapter 4: Test Design Techniques
Equivalence Partitioning (§4.2.1) และ Boundary Value Analysis (§4.2.2)
